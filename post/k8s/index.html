			滚动更新
				strategy:
				  type: RollingUpdate			策略是默认的
				  rollingUpdate:
					maxSurge: 1					新增pod的时候，一次只增加一个
					maxUnavailable: 1			删除旧pod的时候，一次只删除一个	

				实践：
					apiVersion: extensions/v1beta1
					kind: Deployment
					metadata:
					  name: nginx-deploy
					spec:
					  replicas: 3
					  selector:
						matchLabels:
						  app: nginx
					  template:
						metadata:
						  labels:
							app: nginx
						spec:
						  containers:
						  - image: nginx:v1.17.1
							imagePullPolicy: IfNotPresent
							name: nginx				
					实验技巧：
						你找一个本地仓库中没有的镜像

		2.3.6 DaemonSet 
			特点：
				每个节点上都会自动配置一个pod 
			操作：
				pod在定义的时候，必须有labels
				selector 可以忽略
				
			实践：
				apiVersion: extensions/v1beta1
				kind: DaemonSet
				metadata:
				  name: daemonset
				spec:
				  template:
					metadata:
					  name: daemonset
					  labels:
						app: daemonset
					spec:
					  containers:
						- name: daemonset
						  image: nginx:v1.17.2
						  imagePullPolicy: IfNotPresent	
			注意点：
				daemonset 不支持扩容缩容
				更新镜像的时候，仅仅对daemonset生效，而对pod不生效。

To start using your cluster, you need to run the following as a regular user:

  mkdir -p $HOME/.kube
  sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
  sudo chown $(id -u):$(id -g) $HOME/.kube/config

You should now deploy a pod network to the cluster.
Run "kubectl apply -f [podnetwork].yaml" with one of the options listed at:
  https://kubernetes.io/docs/concepts/cluster-administration/addons/

Then you can join any number of worker nodes by running the following on each as root:

kubeadm join 192.168.8.14:6443 --token j1f9ta.7k0h8lr38kp2k8p2 \
    --discovery-token-ca-cert-hash sha256:1dff776372e60fbdecbbfb2e5aaefaa11894f35363051a8db2344e9a2753282b

systemctl enable kubelet

先开启node再开启master，防止node资源未准备好就让master运行；并且如果先启动了master，这时有一个node加入，master会认为集群只有一个node一股脑地将所有服务加入该node，很可能导致集群崩溃。

某个结点node2要从集群删除时，要先将master中的node2信息删除（否则后续不能加入），再将node2的资源拷贝到其他node，再删除node.

kubectl create -f XXX.yaml

kubectl delete pod kube-proxy-4zl5v -n kube-system
